Эта ошибка исходит от призывов к Jacobian и Hessian , а не к minimize . Замена Jacobian(fun) с Jacobian(lambda x: fun(x, a)) и аналогичным образом для Hessian должна сделать трюк (так как теперь дифференцируемая функция имеет только один векторный аргумент).

Еще одна вещь: (a) – это просто, если вы хотите, чтобы это был кортеж (a,) . 

 import numpy as np from scipy.optimize import minimize from numdifftools import Jacobian, Hessian def fun(x, a): return (x[0] - 1) **2 + (x[1] - a) **2 def fun_der(x, a): return Jacobian(lambda x: fun(x, a))(x).ravel() def fun_hess(x, a): return Hessian(lambda x: fun(x, a))(x) x0 = np.array([2, 0]) # initial guess a = 2.5 res = minimize(fun, x0, args=(a,), method='dogleg', jac=fun_der, hess=fun_hess) print(res)

Я понимаю, что это игрушечный пример, но я хотел бы указать, что использование инструмента, такого как Jacobian или Hessian для вычисления производных вместо выведения самой функции достаточно дорого. Например, с помощью вашего метода: 

 x0 = np.array([2, 0]) a = 2.5 %timeit minimize(fun, x0, args=(a,), method='dogleg', jac=fun_der, hess=fun_hess) 100 loops, best of 3: 13.6 ms per loop

Но вы могли бы вычислить производные функции как таковые: 

 def fun_der(x, a): dx = 2 * (x[0] - 1) dy = 2 * (x[1] - a) return np.array([dx, dy] def fun_hess(x, a): dx = 2 dy = 2 return np.diag([dx, dy]) %timeit minimize(fun, x0, args=(a,), method='dogleg', jac=fun_der, hess=fun_hess) 1000 loops, best of 3: 279 µs per loop

Как вы можете видеть, это почти в 50 раз быстрее. Он действительно начинает складываться со сложными функциями. Поэтому я всегда стараюсь самостоятельно вывести функции, независимо от того, насколько это сложно. Один интересный пример – реализация на основе ядра Inductive Matrix Completion . 

 argmin --> sum((A - gamma_u(X) Z gamma_v(Y))**2 - lambda * ||Z||**2) where gamma_u = (1/sqrt(m_x)) * [cos(UX), sin(UX)] and gamma_v = (1/sqrt(m_y)) * [cos(VY), sin(VY)] X.shape = n_x, p; Y.shape = n_y, q; U.shape = m_x, p; V.shape = m_y, q; Z.shape = 2m_x, 2m_y

Вычисление градиента и гессиана из этого уравнения крайне необоснованно по сравнению с явным выводом и использованием этих функций. Так как @bnaul указал, если ваша функция имеет закрытые формы, вы действительно хотите их вычислить и использовать.
